<objective>
Build the `generate-story` Supabase Edge Function — the core generation engine for Phase 6.

Purpose: Accepts a character (sketch) and story theme. Generates 5-page story text via GPT-4o-mini, illustrates each page via gpt-image-1 using the character's animated image as reference, uploads illustrations to Storage, persists everything to the `stories`/`story_pages` tables, and deducts 5 credits on success.
Output: Two Deno TypeScript files — `index.ts` (orchestration) and `story-service.ts` (AI service calls).
</objective>

Create `supabase/functions/generate-story/story-service.ts` with three exported functions:

**`generateStoryText(apiKey, theme, characterDescription)`**
- Calls OpenAI `/v1/chat/completions` with model `gpt-4o-mini`
- System prompt: "You are a children's picture book author. Write a short, joyful, age-appropriate story."
- User prompt: `Write a 5-page children's picture book story. Theme: "${theme}". The main character is described as: "${characterDescription}". Return ONLY valid JSON with this exact shape: { "title": "...", "pages": [ { "page_number": 1, "text": "2-3 sentences of story text for this page.", "scene_description": "one sentence describing what the character is doing in this scene for the illustration." }, ... ] } No markdown, no code blocks, just the raw JSON object.`
- `response_format: { type: "json_object" }`, `max_tokens: 1200`, `temperature: 0.8`
- Returns `{ success: boolean; title?: string; pages?: Array<{ page_number: number; text: string; scene_description: string }>; error?: string }`

**`generatePageIllustration(apiKey, referenceImageBase64, sceneDescription, styleHint)`**
- Takes the character's animated image as base64, a scene description, and style hint (e.g. "Pixar 3D animation style")
- Constructs prompt: `"${styleHint} illustration for a children's picture book. Scene: ${sceneDescription}. Keep the character's exact appearance, colors, and features consistent with the reference image."`
- Calls gpt-image-1 via `/v1/images/edits` with FormData (same pattern as transformImageWithGPTImage1 in openai-service.ts)
- Returns `{ success: boolean; imageBase64?: string; imageUrl?: string; error?: string }`
- On error, returns `{ success: false, error: "..." }` — do NOT throw

**`downloadAndUploadIllustration(supabase, storyId, pageNumber, imageBase64OrUrl)`**
- If result is base64 (starts with `data:image`): strip the data URI prefix, decode with `Uint8Array.from(atob(...))`, upload as ArrayBuffer
- If result is a URL: fetch it, get arrayBuffer, upload
- Storage path: `stories/story_${storyId}_page_${pageNumber}_${Date.now()}.png`
- Bucket: `sketches` (reuse existing bucket — per project pattern)
- Returns the public URL string, or throws on failure

  <action>
Create `supabase/functions/generate-story/index.ts`. Follow the exact same structure as `process-sketch/index.ts`:

**Request body:** `{ sketchId: string, theme: string }`

**Flow:**
1. CORS preflight handling via `handleCorsRequest` / `getCorsHeaders`
2. Validate `Authorization` header (return 401 if missing)
3. Init Supabase client with `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY`
4. Verify user JWT via `supabase.auth.getUser(jwt)`
5. Parse + validate body: require `sketchId` (non-empty string) and `theme` (non-empty string, max 200 chars)
6. Verify sketch ownership: query `sketches` where `id=sketchId AND user_id=userId` — return 403 if not found
7. Fetch sketch row: `SELECT animated_image_url, content_type` — return 400 if `animated_image_url` is null (character hasn't been transformed yet)
8. Check credits: `checkUserCredits(supabase, userId, 5)` — return 402 if insufficient
9. Check budget limits: `checkBudgetLimits(supabase, 5)`
10. Insert a `stories` row with `status='processing'`, `user_id`, `sketch_id=sketchId`, `theme`, `page_count=5` — get back the `storyId`
11. Start async generation (do NOT await — fire and forget using a detached promise):
    ```
    generateAndPersistStory(supabase, storyId, sketchId, theme, animatedImageUrl, contentType, userId, creditCheck.credits, openAIApiKey)
    ```
12. Return immediately: `{ storyId, success: true }` with status 200

**`generateAndPersistStory` function (defined in index.ts, called fire-and-forget):**
- Download the `animated_image_url` image to base64: fetch → arrayBuffer → btoa
- Call `generateStoryText(openAIApiKey, theme, "the main character from the reference image")`
- If story text fails → update story to `status='failed'`, return early (no credit deduction)
- For each of the 5 pages (sequentially):
  - Call `generatePageIllustration(openAIApiKey, characterBase64, page.scene_description, styleHint)`
  - `styleHint` derived from `content_type`: `cartoon` → "vibrant 2D cartoon style", `pixar` → "Pixar 3D animation style", `realistic` → "painterly storybook illustration style"
  - If illustration fails → log warning, set `illustration_url=null` and continue (don't fail the whole story)
  - If illustration succeeded → call `downloadAndUploadIllustration(supabase, storyId, pageNum, result)` to get permanent URL
  - Insert a `story_pages` row: `{ story_id: storyId, page_number: page.page_number, text: page.text, illustration_url: finalUrl }`
- After all 5 pages inserted → update `stories` row: `status='completed', updated_at=NOW()`
- Deduct 5 credits: `deductUserCredits(supabase, userId, currentCredits, 5)`
- Log credit usage via `logCreditUsage(supabase, userId, storyId, 5)` (note: logCreditUsage's 4th param is creditsUsed — pass 5)
- On any uncaught error in the async block → update `stories` to `status='failed'`, log error, do not deduct credits
